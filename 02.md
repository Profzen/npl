## TABLE PAR TABLE — EXPLICATIONS DÉTAILLÉES

### **Table 1 : `branches`** — Les agences physiques

> Représente les agences où travaillent les caissiers. Toute opération (dépôt, retrait) est rattachée à une agence.

```sql
CREATE TABLE branches (
    id INT PRIMARY KEY AUTO_INCREMENT,
    code VARCHAR(20) UNIQUE NOT NULL,
    name VARCHAR(100) NOT NULL,
    city VARCHAR(100) NOT NULL,
    country_code VARCHAR(3) NOT NULL DEFAULT 'SEN',
    address TEXT NULL,
    phone VARCHAR(20) NULL,
    is_active BOOLEAN DEFAULT TRUE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);
```

| Colonne | Explication |
|---------|-------------|
| `id` | Identifiant technique unique |
| `code` | Code métier de l’agence (`AGC-001`) |
| `name` | Nom de l’agence |
| `city` | Ville de l’agence |
| `country_code` | Code pays ISO alpha-3 |
| `address` | Adresse complète libre |
| `phone` | Téléphone de l’agence |
| `is_active` | Agence active ou désactivée |
| `created_at` / `updated_at` | Dates de création et modification |

```sql
    last_login_at TIMESTAMP NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,

```

| `employee_id` | Identifiant de connexion |

### **Table 3 : `customers`** — Les émetteurs

> Le client qui envoie l’argent. Cette table est fortement alignée avec l’objet `sender` de MoneyGram.

```sql
CREATE TABLE customers (
    id INT PRIMARY KEY AUTO_INCREMENT,
    customer_number VARCHAR(20) UNIQUE NOT NULL,
    first_name VARCHAR(100) NOT NULL,
    middle_name VARCHAR(100) NULL,
    last_name VARCHAR(100) NOT NULL,
    second_last_name VARCHAR(100) NULL,
    phone VARCHAR(20) NOT NULL,
    phone_country_dial_code VARCHAR(5) NULL,
    email VARCHAR(150) NULL,
    address_line1 VARCHAR(255) NULL,
    address_line2 VARCHAR(255) NULL,
    city VARCHAR(100) NULL,
    state_province_code VARCHAR(50) NULL,
    postal_code VARCHAR(20) NULL,
    country_code VARCHAR(3) NOT NULL DEFAULT 'SEN',
    date_of_birth DATE NULL,
    birth_country_code VARCHAR(3) NULL,
    nationality_country_code VARCHAR(3) NULL,
    gender ENUM('M','F') NULL,
    id_type VARCHAR(50) NULL,
    id_number VARCHAR(100) NULL,
    id_issue_date DATE NULL,
    id_expiration_date DATE NULL,
    id_issue_country_code VARCHAR(3) NULL,
    occupation VARCHAR(100) NULL,
    source_of_funds VARCHAR(100) NULL,
    politically_exposed_person BOOLEAN DEFAULT FALSE,
    provider_profile_id VARCHAR(100) NULL,
    kyc_level ENUM('basic','standard','enhanced') NOT NULL DEFAULT 'basic',
    is_active BOOLEAN DEFAULT TRUE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);
```

| Colonne | Explication |
|---------|-------------|
| `customer_number` | Identifiant interne du client |
| `first_name`, `middle_name`, `last_name`, `second_last_name` | Nom structuré complet |
| `phone`, `phone_country_dial_code` | Numéro de téléphone + indicatif |
| `email` | Email optionnel |
| `address_line1`, `address_line2`, `city`, `state_province_code`, `postal_code`, `country_code` | Adresse structurée requise par MoneyGram |
| `date_of_birth` | Date de naissance — **obligatoire pour MoneyGram** |

| `occupation` | Profession |
| `source_of_funds` | Origine des fonds |

## 💰 MODÈLE DE FRAIS HYBRIDE

### **Principe**

- **Frais locaux** = estimation rapide au guichet
- **Frais prestataire** = frais réels retournés par MoneyGram
- **Frais client** = `provider_fee + local_fee`

### **Parcours**

1. Le client demande une estimation  
   → on utilise `fee_configurations`

2. Le client confirme qu’il veut envoyer  
   → on appelle l’API MoneyGram pour un `quote`

3. Le client valide  
   → on crée le dépôt à partir du devis

### **Exemple**

| Élément | Valeur |
|---------|--------|
| `provider_fee` | 2 500 XOF |
| `local_fee` | 1 500 XOF |
| `total_fee` | 4 000 XOF |
